## Concept Reference

### What Is a Loss Function?

A loss function $\mathcal{L}(\hat{y}, y)$ measures the discrepancy between the network's prediction $\hat{y}$ and the true target $y$. Training minimises the expected loss over the training distribution:

$$\theta^* = \arg\min_\theta \frac{1}{N} \sum_{i=1}^N \mathcal{L}(f_\theta(\mathbf{x}_i), y_i)$$

The choice of loss function encodes the **probabilistic model** for the data. Specifically, minimising the negative log-likelihood under a particular noise model gives a specific loss function. Mismatching the loss to the data distribution leads to poor performance even with a perfect model architecture.

### Cross-Entropy Loss: Full Derivation

**Probabilistic setup:** Assume the true label $y$ is generated by a categorical distribution parameterised by the network's softmax output. For $K$ classes:

$$P(y = k \mid \mathbf{x}) = \hat{y}_k = \frac{e^{z_k}}{\sum_{j=1}^K e^{z_j}}$$

**Maximum likelihood principle:** Given $N$ training examples, the log-likelihood is:

$$\log P(\mathbf{y} \mid \mathbf{X}, \theta) = \sum_{i=1}^N \log P(y_i \mid \mathbf{x}_i, \theta) = \sum_{i=1}^N \log \hat{y}_{i, y_i}$$

Minimising the negative log-likelihood (NLL) over the training set:

$$\mathcal{L}_{\text{NLL}} = -\frac{1}{N} \sum_{i=1}^N \log \hat{y}_{i, y_i}$$

**One-hot label form:** For a single example with true class $k$ and predicted probabilities $\hat{\mathbf{y}}$:

$$\mathcal{L}_{\text{CE}} = -\sum_{j=1}^K \mathbf{1}[y = j] \log \hat{y}_j = -\log \hat{y}_k$$

Since the label is one-hot, all terms except $j = k$ vanish.

**Connection to KL divergence:**

Let $\mathbf{p}$ be the true distribution (one-hot) and $\hat{\mathbf{y}}$ be the predicted distribution. The cross-entropy is:

$$H(\mathbf{p}, \hat{\mathbf{y}}) = -\sum_k p_k \log \hat{y}_k$$

The KL divergence from $\hat{\mathbf{y}}$ to $\mathbf{p}$ is:

$$\text{KL}(\mathbf{p} \| \hat{\mathbf{y}}) = \sum_k p_k \log \frac{p_k}{\hat{y}_k} = H(\mathbf{p}, \hat{\mathbf{y}}) - H(\mathbf{p})$$

Since $H(\mathbf{p})$ is constant (the entropy of the true distribution does not depend on $\theta$), minimising cross-entropy is equivalent to minimising KL divergence from the true to the predicted distribution.

**Gradient derivation (softmax + cross-entropy):**

For a single example with true class $c$ and logits $\mathbf{z}$:

$$\mathcal{L} = -z_c + \log \sum_k e^{z_k}$$

$$\frac{\partial \mathcal{L}}{\partial z_j} = -\mathbf{1}[j = c] + \frac{e^{z_j}}{\sum_k e^{z_k}} = \hat{y}_j - \mathbf{1}[j = c]$$

In vector form: $\frac{\partial \mathcal{L}}{\partial \mathbf{z}} = \hat{\mathbf{y}} - \mathbf{e}_c$ where $\mathbf{e}_c$ is the one-hot target vector.

The gradient is simply the prediction error: the difference between the predicted probability vector and the target one-hot vector. This is the "clean" result that makes cross-entropy the canonical choice for classification.

### Binary Cross-Entropy (BCE)

For binary classification ($y \in \{0, 1\}$, $\hat{y} = \sigma(z)$):

$$\mathcal{L}_{\text{BCE}} = -y \log \hat{y} - (1 - y) \log(1 - \hat{y})$$

This is a special case of categorical cross-entropy with $K = 2$ and logistic regression as the model. As derived in `backpropagation_derivation.md`, the gradient is $\hat{y} - y$.

**Multi-label binary classification:** When an example can belong to multiple classes simultaneously, apply BCE independently to each label (not softmax cross-entropy, which forces the probabilities to sum to 1). Each output neuron gets its own sigmoid and BCE loss, and the total loss is the sum (or mean) over all labels.

### Mean Squared Error (MSE)

For regression with target $y \in \mathbb{R}$ and prediction $\hat{y} \in \mathbb{R}$:

$$\mathcal{L}_{\text{MSE}} = \frac{1}{N} \sum_{i=1}^N (\hat{y}_i - y_i)^2$$

**Probabilistic derivation:** MSE is the MLE loss under a Gaussian noise model:

$$y_i = f_\theta(\mathbf{x}_i) + \varepsilon_i, \quad \varepsilon_i \sim \mathcal{N}(0, \sigma^2)$$

The likelihood is:

$$P(y_i \mid \mathbf{x}_i, \theta) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\!\left(-\frac{(y_i - \hat{y}_i)^2}{2\sigma^2}\right)$$

Maximising the log-likelihood:

$$\log P = -\frac{N}{2}\log(2\pi\sigma^2) - \frac{1}{2\sigma^2} \sum_{i=1}^N (y_i - \hat{y}_i)^2$$

The only term depending on $\theta$ is $\sum_i (y_i - \hat{y}_i)^2$. Minimising it gives MSE.

**Gradient:** $\frac{\partial \mathcal{L}_{\text{MSE}}}{\partial \hat{y}} = \frac{2}{N}(\hat{y} - y)$

**Properties:**
- Penalises large errors heavily (quadratic penalty)
- Sensitive to outliers: a single large error dominates the loss
- Corresponds to assuming Gaussian-distributed noise around predictions
- Output layer should use linear activation (no sigmoid/tanh): the loss gradient handles scale

### Mean Absolute Error (MAE / L1 Loss)

$$\mathcal{L}_{\text{MAE}} = \frac{1}{N} \sum_{i=1}^N |\hat{y}_i - y_i|$$

**Probabilistic derivation:** MAE is the MLE loss under a Laplace (double-exponential) noise model:

$$P(y \mid \mathbf{x}, \theta) \propto \exp\!\left(-\frac{|y - \hat{y}|}{b}\right)$$

**Gradient:** $\frac{\partial \mathcal{L}_{\text{MAE}}}{\partial \hat{y}} = \text{sign}(\hat{y} - y)$ (undefined at $\hat{y} = y$; typically set to 0)

**Properties:**
- Linear penalty: large errors are penalised proportionally, not quadratically
- Robust to outliers (compared to MSE)
- Non-differentiable at $\hat{y} = y$: gradient is discontinuous (sign function)
- The optimal prediction under MAE is the **median** of the conditional distribution $P(y \mid \mathbf{x})$

### Huber Loss

Huber loss combines MSE and MAE to get robustness without the non-differentiability:

$$\mathcal{L}_{\delta}(\hat{y}, y) = \begin{cases} \frac{1}{2}(\hat{y} - y)^2 & |\hat{y} - y| \leq \delta \\ \delta \left(|\hat{y} - y| - \frac{1}{2}\delta\right) & |\hat{y} - y| > \delta \end{cases}$$

**Derivative:**

$$\frac{\partial \mathcal{L}_\delta}{\partial \hat{y}} = \begin{cases} \hat{y} - y & |\hat{y} - y| \leq \delta \\ \delta \cdot \text{sign}(\hat{y} - y) & |\hat{y} - y| > \delta \end{cases}$$

Huber loss is differentiable everywhere (including at $|\hat{y} - y| = \delta$). For small errors it behaves like MSE (smooth, fast convergence). For large errors it behaves like MAE (linear, robust).

**Choosing $\delta$:** $\delta$ is a hyperparameter defining the threshold between "small" and "large" errors. In practice it is often set to 1.0 or tuned based on the typical residual scale.

**Use cases:** Regression where outliers are expected (e.g., object detection bounding box regression in Faster R-CNN and YOLO).

### Focal Loss

Proposed by Lin et al. (2017) for object detection (RetinaNet) to address extreme class imbalance.

$$\mathcal{L}_{\text{focal}} = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

where:
- $p_t = \hat{y}$ if $y = 1$, else $p_t = 1 - \hat{y}$ (the probability assigned to the correct class)
- $\gamma \geq 0$ is the **focusing parameter** (typically 2)
- $\alpha_t$ is a class-weighting factor (typically $\alpha = 0.25$ for foreground, $1-\alpha$ for background)

**The $(1 - p_t)^\gamma$ term:**

When a sample is easy to classify correctly (high $p_t$, e.g., $p_t = 0.9$):
$$\text{CE loss} = -\log(0.9) \approx 0.105$$
$$\text{Focal loss with } \gamma=2 = -(0.1)^2 \log(0.9) \approx 0.00105$$

The easy example's contribution is down-weighted by $(1 - p_t)^2 = 0.01$, a factor of 100 reduction.

When a sample is hard (low $p_t$, e.g., $p_t = 0.1$):
$$\text{CE loss} = -\log(0.1) \approx 2.303$$
$$\text{Focal loss with } \gamma=2 = -(0.9)^2 \log(0.1) \approx 1.866$$

The hard example's contribution is down-weighted by $(1-0.1)^2 = 0.81$, only a 19% reduction.

**Effect:** Focal loss focuses training on hard examples by down-weighting easy, well-classified examples. In a dataset with 1000 background patches for every foreground object, CE loss is dominated by easy background examples. Focal loss suppresses their contribution, allowing the network to focus on the rare, hard foreground objects.

### Contrastive Loss

Used in metric learning to train embeddings such that similar pairs are close and dissimilar pairs are far apart (Hadsell et al. 2006).

For a pair $(i, j)$ with embeddings $\mathbf{e}_i, \mathbf{e}_j$ and label $y_{ij} \in \{0, 1\}$ (1 = similar, 0 = dissimilar):

$$d_{ij} = \|\mathbf{e}_i - \mathbf{e}_j\|_2$$

$$\mathcal{L}_{\text{contrastive}} = y_{ij} \cdot d_{ij}^2 + (1 - y_{ij}) \cdot \max(0, m - d_{ij})^2$$

where $m$ is a margin hyperparameter.

**Interpretation:**
- If $y_{ij} = 1$ (similar): loss is $d_{ij}^2$. Minimised when similar pairs are close ($d_{ij} \to 0$).
- If $y_{ij} = 0$ (dissimilar): loss is $\max(0, m - d_{ij})^2$. Zero when $d_{ij} \geq m$ (pairs are already separated by at least the margin). Penalises when they are too close.

**Triplet loss (related):** Uses anchor $a$, positive $p$, and negative $n$ examples:

$$\mathcal{L}_{\text{triplet}} = \max\!\left(0,\ d(a, p) - d(a, n) + \text{margin}\right)$$

Pushes the anchor closer to positive examples than to negative examples by at least the margin.

### Loss Function Summary

| Loss | Data type | Noise model | Sensitive to outliers | Output activation |
|---|---|---|---|---|
| BCE | Binary classification | Bernoulli | No | Sigmoid |
| Categorical CE | Multi-class classification | Categorical | No | Softmax |
| MSE | Regression | Gaussian | Yes | Linear |
| MAE | Regression | Laplace | No | Linear |
| Huber | Regression | Gaussian + clipped | Moderate | Linear |
| Focal | Imbalanced classification | Categorical + down-weighting | No | Sigmoid |
| Contrastive | Metric learning | N/A | N/A | $\ell_2$ normalised |

## Tier 1 -- Fundamentals

### Question F1
**Derive binary cross-entropy loss from the principle of maximum likelihood estimation. Clearly state the probabilistic model being assumed.**

**Answer:**

**Probabilistic model:**

Assume each label $y_i \in \{0, 1\}$ is drawn from a Bernoulli distribution with parameter $\hat{y}_i = \sigma(f_\theta(\mathbf{x}_i))$:

$$P(y_i = 1 \mid \mathbf{x}_i, \theta) = \hat{y}_i, \quad P(y_i = 0 \mid \mathbf{x}_i, \theta) = 1 - \hat{y}_i$$

In compact form: $P(y_i \mid \mathbf{x}_i, \theta) = \hat{y}_i^{y_i}(1 - \hat{y}_i)^{1 - y_i}$

**Joint likelihood over $N$ i.i.d. training examples:**

$$\mathcal{L}_{\text{MLE}}(\theta) = \prod_{i=1}^N P(y_i \mid \mathbf{x}_i, \theta) = \prod_{i=1}^N \hat{y}_i^{y_i}(1 - \hat{y}_i)^{1-y_i}$$

**Log-likelihood (more tractable):**

$$\log \mathcal{L}_{\text{MLE}}(\theta) = \sum_{i=1}^N \left[y_i \log \hat{y}_i + (1 - y_i) \log(1 - \hat{y}_i)\right]$$

**Negative log-likelihood (to minimise):**

$$\mathcal{L}_{\text{BCE}}(\theta) = -\frac{1}{N} \sum_{i=1}^N \left[y_i \log \hat{y}_i + (1 - y_i) \log(1 - \hat{y}_i)\right]$$

**The factor $\frac{1}{N}$:** Dividing by $N$ converts the total log-likelihood to a per-example average, making the loss scale invariant to dataset size and allowing the same learning rate to be used regardless of $N$.

**Connection to entropy:** For a true label distribution $p \in \{0,1\}$ and predicted $\hat{y}$, the BCE is the cross-entropy $H(p, \hat{y}) = -p \log \hat{y} - (1-p) \log(1-\hat{y})$, which measures the encoding cost of the predicted distribution when the true distribution is $p$.

### Question F2
**When should you use MSE vs MAE vs Huber loss for a regression problem? Give concrete examples.**

**Answer:**

**Use MSE when:**
- Errors are approximately Gaussian
- Large errors are genuinely important and should be penalised heavily
- You want the optimal prediction to be the conditional mean $\mathbb{E}[y \mid \mathbf{x}]$

*Example:* Predicting house prices from features where the data is clean and errors are roughly symmetric. A house that should cost \$300k being predicted at \$500k (off by \$200k) should penalise more than 4 predictions each off by \$50k, which MSE captures.

**Use MAE when:**
- There are outliers in the labels (corrupted data, labelling errors)
- You want the optimal prediction to be the conditional median $\text{median}(y \mid \mathbf{x})$, which is more robust than the mean
- You can tolerate the non-differentiability at zero residuals (or use a smooth approximation)

*Example:* Predicting delivery times where some labels are corrupted (driver forgot to close the app, recording 8-hour "deliveries"). These outliers would dominate MSE but barely affect MAE.

**Use Huber loss when:**
- You want robustness to occasional outliers but still want fast convergence on clean examples
- The transition between "small" and "large" error is meaningful and can be tuned
- Differentiability everywhere is important (e.g., second-order optimisation)

*Example:* Bounding box regression in object detection. Most anchors are close to their target box (small residuals), but some are very far (large residuals when the anchor is poorly placed). Huber loss converges quickly for the close anchors and doesn't let the far ones destabilise training.

**Comparison at a glance:**

```
Error = 1.0: MSE = 1.0,  MAE = 1.0,  Huber (δ=1) = 0.5
Error = 5.0: MSE = 25.0, MAE = 5.0,  Huber (δ=1) = 4.5
Error = 10.0: MSE = 100, MAE = 10.0, Huber (δ=1) = 9.5

MSE heavily penalises the error=10 case (100 vs 10 for MAE).
Huber behaves like MAE for large errors (linear) and MSE for small (quadratic).
```

### Question F3
**Why is it wrong to use MSE loss for a classification task with sigmoid output? What happens to the gradients?**

**Answer:**

**The mathematical problem:**

For binary classification with sigmoid output $\hat{y} = \sigma(z)$ and MSE loss:

$$\mathcal{L}_{\text{MSE}} = \frac{1}{2}(\hat{y} - y)^2$$

The gradient with respect to the logit $z$ is:

$$\frac{\partial \mathcal{L}_{\text{MSE}}}{\partial z} = (\hat{y} - y) \cdot \sigma'(z) = (\hat{y} - y) \cdot \hat{y}(1-\hat{y})$$

**When the network is confidently wrong:**

Suppose $y = 0$ (true label) and the network outputs $\hat{y} = 0.99$ (very confident wrong prediction). The prediction error $\hat{y} - y = 0.99$. But the sigmoid derivative is $\sigma'(z) = 0.99 \times 0.01 = 0.0099$.

$$\frac{\partial \mathcal{L}_{\text{MSE}}}{\partial z} = 0.99 \times 0.0099 \approx 0.0098$$

This is a near-zero gradient despite a large error! The saturated sigmoid kills the gradient.

**Contrast with cross-entropy:**

For the same scenario, the cross-entropy gradient is:

$$\frac{\partial \mathcal{L}_{\text{CE}}}{\partial z} = \hat{y} - y = 0.99$$

The gradient is 0.99 -- large, reflecting the large error.

**Why cross-entropy is the right choice:**

Cross-entropy "un-saturates" the sigmoid by design. The $\hat{y}(1-\hat{y})$ factor in the sigmoid derivative is exactly cancelled by the $\frac{1}{\hat{y}(1-\hat{y})}$ factor in the cross-entropy gradient, giving a clean error signal. MSE does not have this cancellation.

**Additional problem: non-convexity.** MSE + sigmoid creates a loss surface with multiple local minima. Cross-entropy + sigmoid (with a linear last layer) creates a convex loss in the logit space.

## Tier 2 -- Intermediate

### Question I1
**Explain the class imbalance problem in classification and describe three strategies to address it at the loss function level. Derive the modified loss for each strategy.**

**Answer:**

**The class imbalance problem:**

Suppose 99% of examples are class 0 and 1% are class 1. A model that always predicts class 0 achieves 99% accuracy. Cross-entropy loss for such a model:

$$\mathcal{L} = -\frac{1}{N}\left(N_0 \log(1) + N_1 \log(0^+)\right) \approx -\frac{N_1}{N} \log(p_1)$$

The loss is dominated by the $N_0$ easy negatives, providing almost no gradient signal from the rare positives. The network converges to always predicting the majority class.

**Strategy 1: Weighted cross-entropy**

Assign a weight $w_k$ to each class:

$$\mathcal{L}_{\text{weighted}} = -\frac{1}{N} \sum_{i=1}^N w_{y_i} \log \hat{y}_{i, y_i}$$

Common choice: $w_k = \frac{N}{K \cdot N_k}$ (inverse class frequency, where $N_k$ is the count of class $k$ and $K$ is the number of classes). For binary case with $w_1 = \frac{N}{2 N_1}$ and $w_0 = \frac{N}{2 N_0}$:

$$\mathcal{L}_{\text{weighted BCE}} = -\frac{1}{N}\sum_{i=1}^N \left[w_1 y_i \log \hat{y}_i + w_0 (1-y_i) \log(1-\hat{y}_i)\right]$$

With 1% positive: $w_1 = 50$, $w_0 \approx 0.505$. Each positive example contributes 50 times more to the loss than a negative example, compensating for the imbalance.

**Strategy 2: Focal loss**

$$\mathcal{L}_{\text{focal}} = -\frac{1}{N} \sum_{i=1}^N \alpha_{y_i}(1 - p_{t_i})^\gamma \log(p_{t_i})$$

where $p_{t_i} = \hat{y}_{i, y_i}$ (probability of the correct class) and $\gamma > 0$.

The $(1 - p_{t_i})^\gamma$ factor down-weights easy examples (high $p_{t_i}$) and focuses training on hard examples. For $\gamma = 2$ and $p_t = 0.9$: down-weight by $(0.1)^2 = 0.01$. For $p_t = 0.1$: down-weight by $(0.9)^2 = 0.81$.

**Strategy 3: Label smoothing**

Replace the hard one-hot target $y$ with a smoothed target $\tilde{y} = (1 - \varepsilon) y + \varepsilon / K$:

$$\mathcal{L}_{\text{smooth}} = -\sum_{k=1}^K \tilde{y}_k \log \hat{y}_k = (1-\varepsilon) \mathcal{L}_{\text{CE}} + \frac{\varepsilon}{K} \sum_k (-\log \hat{y}_k)$$

For binary case ($\varepsilon = 0.1$): targets become $\tilde{y} = 0.9$ for positives and $\tilde{y} = 0.1$ for negatives. This prevents the model from becoming overconfident in either class. It acts as a regulariser and reduces the problem of the model learning to exploit hard thresholds, which often hurts minority class recall.

**Which to use:** Focal loss is generally most effective for extreme imbalance (e.g., object detection). Weighted cross-entropy is simpler and works well for moderate imbalance. Label smoothing is primarily a regulariser rather than an imbalance fix and is best combined with one of the other approaches.

### Question I2
**Contrastive learning requires carefully designed loss functions. Compare contrastive loss, triplet loss, and NT-Xent (InfoNCE) loss. What problem does NT-Xent solve that triplet loss does not?**

**Answer:**

**Contrastive loss (pairwise):**

$$\mathcal{L}_{\text{contrastive}} = y_{ij} d_{ij}^2 + (1-y_{ij}) \max(0, m - d_{ij})^2$$

*Problem:* Requires pre-defined positive/negative pairs. For $N$ training examples, generating $O(N^2)$ pairs is expensive. Many pairs are uninformative (easy negatives that are already well-separated by the margin $m$).

**Triplet loss:**

$$\mathcal{L}_{\text{triplet}} = \max\!\left(0, d(a, p) - d(a, n) + m\right)$$

*Improvement:* Uses relative comparisons (anchor should be closer to positive than to negative by margin $m$). More informative than pairwise comparisons.

*Problem:* **Hard negative mining** is critical but difficult. Most randomly sampled triplets are easy (negative already far from anchor) and produce zero gradient. The network only learns from the rare hard triplets where $d(a, n) < d(a, p) + m$. Without careful mining, training is very slow. Mining too aggressively (using the hardest negatives) can cause training instability.

**NT-Xent (Normalised Temperature-Scaled Cross-Entropy, InfoNCE):**

For a batch of $N$ pairs (each sample augmented twice), with augmented pair $(i, j)$ being the positive pair:

$$\mathcal{L}_{i} = -\log \frac{\exp(\text{sim}(\mathbf{z}_i, \mathbf{z}_j) / \tau)}{\sum_{k=1}^{2N} \mathbf{1}[k \neq i] \exp(\text{sim}(\mathbf{z}_i, \mathbf{z}_k) / \tau)}$$

where $\text{sim}(\mathbf{u}, \mathbf{v}) = \frac{\mathbf{u}^{\top} \mathbf{v}}{\|\mathbf{u}\| \|\mathbf{v}\|}$ is cosine similarity and $\tau$ is the temperature hyperparameter.

**What NT-Xent solves that triplet loss does not:**

1. **Implicit hard negative mining at scale.** Every example in the batch serves as a negative for every other pair. With a batch of 4096 examples (common in SimCLR), each positive pair is contrasted against 4094 negatives simultaneously. Harder negatives that are close to the anchor in embedding space automatically receive higher weight in the softmax denominator, providing a hard-negative mining effect without explicit mining.

2. **Calibrated comparisons via temperature.** The temperature $\tau$ controls the sharpness of the distribution over negatives. Low $\tau$ sharpens the distribution (focuses on the hardest negatives). High $\tau$ softens it. This provides a principled way to control the difficulty of the learning signal.

3. **No margin hyperparameter.** Triplet loss requires tuning the margin $m$, which is task-dependent. NT-Xent has only $\tau$, which is easier to set (often 0.07 to 0.5) and has a cleaner theoretical interpretation as an inverse temperature.

4. **Mutual information interpretation.** NT-Xent maximises a lower bound on the mutual information between two views of the same example. This provides principled justification for the loss beyond the geometric intuition of triplet loss.

## Tier 3 -- Advanced

### Question A1
**The KL divergence $\text{KL}(p \| q)$ is not symmetric. For knowledge distillation and variational autoencoders, practitioners must choose the direction of the divergence. Derive the different gradients and explain what each direction optimises for.**

**Answer:**

The KL divergence from distribution $q$ to distribution $p$ is:

$$\text{KL}(p \| q) = \sum_x p(x) \log \frac{p(x)}{q(x)} = \mathbb{E}_p\left[\log p - \log q\right]$$

The reverse direction:

$$\text{KL}(q \| p) = \sum_x q(x) \log \frac{q(x)}{p(x)} = \mathbb{E}_q\left[\log q - \log p\right]$$

**Forward KL: $\text{KL}(p \| q)$ -- minimising w.r.t. $q$**

$$\min_q \text{KL}(p \| q) = \min_q -\sum_x p(x) \log q(x) + \text{const}$$

This is minimised by maximising $\mathbb{E}_p[\log q(x)]$ -- the expected log-likelihood of $q$ under samples from $p$.

*Mass-covering behaviour:* $\text{KL}(p \| q)$ is infinite when $p(x) > 0$ and $q(x) = 0$. The optimised $q$ must cover all regions where $p$ has mass. If $p$ is multi-modal, $q$ will spread out to cover all modes, even if it puts probability mass in low-probability valleys between modes.

**Reverse KL: $\text{KL}(q \| p)$ -- minimising w.r.t. $q$**

$$\min_q \text{KL}(q \| p) = \min_q \sum_x q(x) \log q(x) - \sum_x q(x) \log p(x)$$

$$= \min_q -H(q) - \mathbb{E}_q[\log p]$$

*Mode-seeking behaviour:* $\text{KL}(q \| p)$ is finite even when $q(x) > 0$ but $p(x) \approx 0$ -- it is penalised but not infinite. However, when $q(x) > 0$ and $p(x) = 0$, the term $q(x) \log(q(x)/p(x)) = +\infty$. Therefore $q$ is strongly penalised for placing mass where $p$ has zero mass. Optimised $q$ avoids regions where $p$ is small: it seeks a single mode of $p$.

**Application in knowledge distillation:**

Student network $q_\theta$, teacher network $p$ (fixed).

- **Forward KL $\text{KL}(p \| q_\theta)$:** Student covers all teacher modes. If the teacher assigns non-trivial probability to multiple classes, the student must also spread its probability. Appropriate when you want the student to match the teacher's full predictive uncertainty.

- **Reverse KL $\text{KL}(q_\theta \| p)$:** Student locks onto the teacher's most probable output. May ignore tail probabilities. This is equivalent to maximum likelihood from teacher soft labels -- the standard approach. In practice, the knowledge distillation loss (Hinton et al. 2015) is:

$$\mathcal{L}_{\text{KD}} = \tau^2 \text{KL}(p_\tau \| q_\tau)$$

where $p_\tau$ and $q_\tau$ are teacher and student softmax outputs at temperature $\tau > 1$, which softens the teacher distribution to reveal its dark knowledge (relative class similarities).

**Application in VAEs:**

The ELBO (Evidence Lower Bound) objective is:

$$\text{ELBO} = \mathbb{E}_{q(\mathbf{z} \mid \mathbf{x})}[\log p(\mathbf{x} \mid \mathbf{z})] - \text{KL}(q(\mathbf{z} \mid \mathbf{x}) \| p(\mathbf{z}))$$

The second term is $\text{KL}(q \| p)$ -- reverse KL from the approximate posterior $q$ to the prior $p$. This is mode-seeking: the learned posterior $q$ is penalised for placing mass in regions the prior $p$ assigns near-zero probability. The prior $p(\mathbf{z}) = \mathcal{N}(0, \mathbf{I})$ constrains the latent space to be compact and smooth. Using forward KL $\text{KL}(p \| q)$ would require sampling from the true posterior $p(\mathbf{z} \mid \mathbf{x})$, which is intractable -- this is precisely why VAEs use reverse KL and the reparameterisation trick.

### Question A2
**Derive the gradient of the cross-entropy loss with respect to the logits $\mathbf{z}$ for multi-class classification. Show that the Jacobian of the softmax is not needed explicitly, and explain why implementations compute this gradient directly rather than via the chain rule through the Jacobian.**

**Answer:**

**Setup:** $K$ classes, logits $\mathbf{z} \in \mathbb{R}^K$, softmax output $\hat{y}_k = \frac{e^{z_k}}{\sum_j e^{z_j}}$, true class $c$ (one-hot target $\mathbf{y}$ with $y_c = 1$, $y_{k \neq c} = 0$).

**Cross-entropy loss:**

$$\mathcal{L} = -\sum_k y_k \log \hat{y}_k = -\log \hat{y}_c = -z_c + \log \sum_j e^{z_j}$$

The second equality uses $\log \hat{y}_c = \log e^{z_c} - \log \sum_j e^{z_j} = z_c - \log \sum_j e^{z_j}$.

**Direct gradient with respect to $z_k$:**

For $k = c$:
$$\frac{\partial \mathcal{L}}{\partial z_c} = -1 + \frac{e^{z_c}}{\sum_j e^{z_j}} = -1 + \hat{y}_c = \hat{y}_c - 1$$

For $k \neq c$:
$$\frac{\partial \mathcal{L}}{\partial z_k} = 0 + \frac{e^{z_k}}{\sum_j e^{z_j}} = \hat{y}_k$$

Combining: $\frac{\partial \mathcal{L}}{\partial z_k} = \hat{y}_k - y_k$ for all $k$.

In vector form: $\nabla_{\mathbf{z}} \mathcal{L} = \hat{\mathbf{y}} - \mathbf{y}$

**Why the Jacobian route is avoided:**

The naive chain rule approach would compute:

$$\nabla_{\mathbf{z}} \mathcal{L} = \mathbf{J}_{\text{softmax}}^{\top} \nabla_{\hat{\mathbf{y}}} \mathcal{L}$$

where $\mathbf{J}_{\text{softmax}} \in \mathbb{R}^{K \times K}$ has entries $J_{ij} = \hat{y}_i(\delta_{ij} - \hat{y}_j)$.

And $\nabla_{\hat{\mathbf{y}}} \mathcal{L}$ has entries $\frac{\partial \mathcal{L}}{\partial \hat{y}_k} = -\frac{y_k}{\hat{y}_k}$.

Computing this requires:
1. Computing the full $K \times K$ Jacobian matrix
2. Multiplying it by the $K$-vector gradient

The Jacobian computation costs $O(K^2)$ memory and $O(K^2)$ operations.

**The direct formula costs $O(K)$.** For ImageNet with $K = 1000$ classes, this is a factor of 1000 savings per training example. For language model vocabularies with $K = 50,000$, the Jacobian would be a $50,000 \times 50,000$ matrix -- 2.5 billion entries, completely impractical.

**Why the simplification works:**

The cross-entropy and softmax pair is a natural exponential family pairing. The loss function is specifically designed to pair with softmax. The $(1/\hat{y}_k)$ in the CE gradient cancels the $\hat{y}_k(1 - \hat{y}_k)$ in the softmax diagonal Jacobian, and the cross-terms also collapse cleanly. This is the same structural cancellation that occurs with sigmoid + binary cross-entropy.

This is why deep learning frameworks implement `cross_entropy_loss(logits, targets)` as a single fused operation rather than `nll_loss(softmax(logits), targets)` -- they directly compute $\hat{\mathbf{y}} - \mathbf{y}$ in the backward pass without ever materialising the full Jacobian.
